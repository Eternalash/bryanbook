1. #### 时间类型查询效率\(如果需要对时间字段进行操作，推荐使用bigint，如果时间字段不需要进行任何操作，推荐使用timestamp，使用4个字节保存比较节省空间，但是只能记录到2038年\)：

   1. 在InnoDB存储引擎下，通过时间范围查找性能：bigint&gt;datetime&gt;timestamp

   2. 在InnoDB存储引擎下，分组速率：timestamp&gt;datetime （bigint不能用于分组，每个数值都会一个组）

   3. 在InnoDB存储引擎下，排序速率:bigint&gt;timestamp&gt;datetime
2. 数据类型

   1. | 类型 | 简介 | 特性 | 场景 |
      | :--- | :--- | :--- | :--- |
      | String\(字符串\) | 二进制安全 | 可以包含任何数据,比如jpg图片或者序列化的对象,一个键最大能存储512M |  |
      | Hash（字典） | 键值对集合，即编程语言中的Map类型 | 适合存储对象,并且可以像数据库中update一个属性一样只修改某一项属性值\(Memcached中需要取出整个字符串反序列化成对象修改完再序列化存回去\) | 存储、读取、修改用户属性 |
      | List（列表） | 链表（双向链表） | 增删快，提供了操作某一段元素的API | 1.最新消息排行等功能（朋友圈的时间线）2.消息队列 |
      | Set（集合） | 哈希表实现，元素不重复 | 1.添加删除查找的复杂度都是O\(1\)2.为集合提供了求交集、并集、差集等操作 | 1.共同好友2.利用唯一性，统计访问网站的所有独立ip2.好友推荐时，根据tag求交集，大于某个阈值就可以推荐 |
      | SortedSet（有序集合） | 将set中的元素增加一个权重参数score，元素按照score有序排序 | 数据拆入集合时，已经进行天然排序 | 1.排行榜2.带权重的消息队列 |
      | HyperLogLog | 用来做基数统计 |  |  |
